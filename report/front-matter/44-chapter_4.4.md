## 4.4. Web Applications UX/UI Design
El diseño UX/UI de la aplicación web se enfoca en brindar una experiencia clara, eficiente y fácil de usar para la gestión y monitoreo de sistemas hídricos. Desde la perspectiva UX, se prioriza la organización de la información y los flujos de navegación, permitiendo a los usuarios identificar rápidamente el estado del sistema, detectar anomalías y tomar decisiones oportunas. En cuanto al diseño UI, se emplea una interfaz limpia y estructurada, con elementos visuales como indicadores, botones y paneles que facilitan la interacción. En conjunto, el diseño busca combinar funcionalidad y simplicidad para mejorar la toma de decisiones operativas en tiempo real.

### 4.4.2. Web Applications Wireflow Diagrams
Los Wireflow Diagrams de la aplicación web representan de manera visual los flujos de navegación y la interacción del usuario dentro del sistema. Estos diagramas combinan wireframes y flujos de proceso para mostrar cómo el usuario accede, monitorea el estado del sistema hídrico, detecta incidencias y ejecuta acciones correctivas. Su uso permite identificar posibles problemas de usabilidad y asegurar una experiencia coherente y eficiente. En el caso de Aquanetix, el wireflow evidencia el flujo principal desde el inicio de sesión hasta la gestión de alertas, así como accesos secundarios a funcionalidades como configuración, notificaciones y análisis de datos.
